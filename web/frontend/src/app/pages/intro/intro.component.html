What are

Objects have protocols

Think of a file handle. You can't read before opening. You can't close twice. There's a sequence of operations you must follow.

A session type describes this protocol formally — which methods are available, in what order, with what choices.

In our notation: open . &{read: close . end, write: close . end}

The &{...} is a branch — the environment chooses which method to call.

1 open() method
2 read() or write() branch
3 close() method
4 end done

Types become state spaces

Each session type unfolds into a state space — a directed graph where nodes are states and edges are method calls.

The top node is the initial state (before any call). The bottom is the terminal state (end).

Reachability defines a partial order: state A ≥ state B if there's a path from A to B. This gives us a poset.

Posets become lattices

A poset is a lattice if every pair of elements has a meet (greatest lower bound) and a join (least upper bound).

We prove that every well-formed session type's state space is a bounded lattice. The top is the initial state. The bottom is end. Every pair has a meet and join.

This is the Reticulate Theorem — proved constructor by constructor, mechanised in Lean 4 with zero sorry.

✓ Has unique top (initial state)
✓ Has unique bottom (end state)
✓ All pairs have a meet (s₁ ∧ s₂)
✓ All pairs have a join (s₁ ∨ s₂)
≡ Bounded lattice — a reticulate

Parallel forces products

The parallel constructor (∥) models two concurrent paths on a shared object. Both must complete before the session continues.

The state space of S₁ ∥ S₂ is the product lattice L(S₁) × L(S₂), ordered componentwise. Products of lattices are lattices.

This makes lattice structure necessary — not merely a nice property, but an inevitable consequence of concurrency.

L(S₁ ∥ S₂) = L(S₁) × L(S₂)

The Reticulate Theorem

For every well-formed session type S, the SCC quotient of its state space, ordered by reachability, is a bounded lattice.

Proved by structural induction on constructors. Mechanised in Lean 4 (zero sorry). Verified on 34 benchmarks.
