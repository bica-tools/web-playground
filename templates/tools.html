Reticulate — Python CLI & Library

Reticulate is a Python library and command-line tool that constructs state spaces from session type definitions, checks whether they form lattices, and generates Hasse diagrams. It implements the full theory: parsing, state-space construction (including the product construction for the parallel constructor ∥), SCC-based lattice checking, morphism classification, termination analysis, and JUnit 5 test generation.

Features

  • Session type parser with Unicode support (⊕, ∥, μ)
  • State-space construction with product lattice for parallel (∥)
  • Lattice checking via SCC quotient and pairwise meet/join verification
  • Morphism hierarchy: isomorphism, embedding, projection, Galois connection
  • Termination and well-formedness checking for recursive parallel types
  • Hasse diagram generation (DOT/Graphviz)
  • JUnit 5 test generation from session type state spaces

Requirements

  • Python 3.11+
  • Optional: graphviz (Python package + system binary) for diagram rendering

Installation

git clone https://github.com/alcides/bica.git
cd bica/reticulate
python3 -m pytest tests/ -v   # verify installation

Quick Start

Parse and check a simple branch type:

$ python3 -m reticulate "&{read: end, write: end}"
Type:        &{read: end, write: end}
States:      3
Transitions: 2
SCCs:        3
Lattice:     Yes

Check a recursive type with parallel:

$ python3 -m reticulate "(rec X . &{next: X, done: end} || a.b.end)"
Type:        (rec X . &{next: X, done: end} || a.b.end)
States:      6
Transitions: 8
SCCs:        6
Lattice:     Yes

Generate a Hasse diagram:

$ python3 -m reticulate --hasse --fmt svg "init . &{open: close.end, fail: end}"

BICA Reborn — Java Annotation Processor

BICA Reborn is a Java annotation-based session type checker for objects, the successor to the original BICA (2009). It verifies that Java classes conform to session type protocols at compile time via @Session annotations. Key novelty: the ∥ constructor for concurrent access, which forces lattice structure.

Features

  • Session type annotations: @Session, @Shared, @ReadOnly, @Exclusive
  • Compile-time typestate checking via annotation processor
  • Concurrency classification: shared, sync, read-only, exclusive
  • Thread safety verification based on concurrency lattice
  • Standalone CLI tool for analysis without compilation
  • JUnit 5 test generation from @Session annotations
  • All 34 benchmark protocols verified

Requirements

  • Java 21+
  • Maven 3

Installation

git clone https://github.com/alcides/bica.git
cd bica/bica
mvn install        # build and install to local repo
mvn test           # run 1,024 tests

Usage: Annotation

Annotate your Java class with a session type:

import com.bica.reborn.annotation.Session;

@Session("init . &{open: +{OK: read.close.end, ERROR: end}}")
public class FileHandle {
    public void init() { /* ... */ }
    public void open() { /* ... */ }
    public boolean read() { /* ... */ }
    public void close() { /* ... */ }
}

Usage: CLI

$ java -jar bica-reborn.jar "&{read: end, write: end}"
Type:        &{read: end, write: end}
States:      3
Transitions: 2
Lattice:     Yes

Maven Dependency

<dependency>
    <groupId>com.bica</groupId>
    <artifactId>bica-reborn</artifactId>
    <version>0.1.0-SNAPSHOT</version>
</dependency>
