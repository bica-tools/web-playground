The Lattice Is the Board

Session types and game theory share deep structural connections.

🎯

Two Players

Client controls external choice (&{...}). Server controls internal choice (+{...}). Every protocol is a two-player game.

🏁

Strategies Are Paths

A strategy is a path from ⊤ to ⊥ through the Hasse diagram. The lattice guarantees every game terminates with a well-defined outcome.

⚖

Meet & Join

Meet is where two execution paths converge. Join is where they split. The lattice structure guarantees these always exist.

🔄

Recursion = Loops

Recursive types create cycles in the game tree. The game can loop — but termination analysis guarantees it will eventually reach ⊥.

Explore More

Ideas for interactive explorations of session type theory.

🔍

Find the Meet

Given two states in a lattice, find their greatest lower bound. A puzzle that builds intuition for lattice operations.

🚨

Spot the Violation

A sequence of method calls is shown. Is it a valid protocol execution? Find the first illegal move.

🧩

Build a Protocol

Drag-and-drop branches, selections, and recursion to construct a session type. Watch the Hasse diagram build in real time.

🌊

Coverage Race

Given a protocol, find the minimum set of test paths that cover all transitions. Beat the generated test suite!

🔮

Dual Mirror

Play as the Client while your reflection plays the dual type. Every branch you take, your mirror takes the opposite selection.

🎭

Multiparty Arena

Three or more players, each controlling a projected role from a global type. Coordinate to complete the protocol.
